# Contributing to SyncMind 

Thank you for your interest in contributing to SyncMind! We welcome contributions from everyone and appreciate your effort in improving the project.

## Local Development Setup

### 1. Fork the repository

Click the "Fork" button on GitHub to create your own copy of the repository.

### 2. Clone your fork

```bash
git clone https://github.com/YOUR_USERNAME/SyncMind.git
cd SyncMind
```

### 3. Install dependencies

```bash
npm install
```

### 4. Run the development server

```bash
npm run dev
```

### 5. Open in browser

```bash
http://localhost:5173
```

---

## Code Style Guidelines

- Follow the existing folder structure
- Use meaningful variable and function names
- Keep components reusable and modular
- Remove unnecessary console logs
- Maintain clean formatting and proper indentation
- Write readable and maintainable code

---

## Branch Naming Convention

Create a new branch before making changes.

Examples:

```bash
feat/add-authentication
fix/navbar-responsive-bug
docs/update-contributing-guide
refactor/improve-api-handling
```

---

## Commit Message Format

Use meaningful commit messages.

Examples:

```bash
feat: add authentication feature
fix: resolve navbar issue
docs: add contributing guide
refactor: optimize API requests
```

---

## Pull Request Process

1. Fork the repository
2. Create a new branch
3. Make changes
4. Commit your changes
5. Push your branch

```bash
git push origin your-branch-name
```

6. Open a Pull Request
7. Add a proper title and description
8. Link the related issue if applicable

---

## Issue Reporting Guidelines

Please include:

- Clear issue description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Browser and operating system details

---

## Code of Conduct

Please be respectful and maintain a welcoming environment for all contributors.
